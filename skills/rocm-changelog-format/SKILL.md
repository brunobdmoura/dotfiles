---
name: rocm-changelog-format
description: Generate a debian/changelog entry using `dch` and populate it with the recent git commit history.
---

## Inputs
- `$PACKAGE_NAME` - The name of the package being bumped.
- `$VERSION` - The numeric string containing the new upstream version, e.g., `7.2.3`.
- `$RELEASE` - The target release name, e.g., `resolute`. Defaults to `stonking` if not provided.

## Step 1 — Gather git context

1. Navigate: If not already inside a directory named `$PACKAGE_NAME`, step inside it.
2. Read `debian/changelog` to determine the suffix: Check if previous changelog entries have derived name strings (e.g., `+ds`, `+dfsg`). If so, save this suffix to append to the `$VERSION` value.
3. Find the previous version: Determine the git tag or commit hash of the last release.
4. Find the <last_commit_hash> refering to a changelog entry on its message by running `git log --oneline` and parsing the commit hash at the start of the line.
4. Get commit history: Run `git log <last_commit_hash>..HEAD --pretty=format:"%an|%s"` to get the author name and commit message for all new commits since the last changelog bump.

## Step 2 - Prepare changelog entry

If there is no changelog entry matching `$VERSION`, run the `dch` command to create the initial entry with the exact version string and distribution;
use the suffix found in Step 1 (if any): `dch -v "$VERSION<suffix>-0ubuntu1" --distribution "$RELEASE" "Update changelog"`. If there is already an entry of that
version, prepare to compose its body on Step 3.

## Step 3 — Compose changelog entry

Modify the top block in `debian/changelog`. Group the commits parsed in Step 1 by their author, ordering the commits from oldest to newest.
If there is only a single author, the dedicated line with their name can be removed.

Obey the following formatting rules strictly:
- The changelog title must follow standard Debian syntax.
- Each line below the changelog title must have `2` blank spaces before the commit group name or the commit message bullet points.
- Wrap lines at 70 characters. Do not split words. If wrapping to a newline, pad the start of the newline with `4` blank spaces to align with the previous text.
- Preserve the signature/trailer line (` -- Name <email>  <date>`) generated by `dch` at the bottom of the block.

An example result of a correctly formatted changelog entry block:

```txt
hipsparselt (7.2.4+ds-0ubuntu1) stonking; urgency=medium

  [ Talha Can Havadar ]
  * d/{salsa-ci.yml, ci/}: update salsa-ci for ubuntu

  [ Bruno Bernardo de Moura ]
  * d/watch: update file to target monorepo
  * New upstream version 7.2.4
  * d/{hipsparselt-dev, libhipsparselt0}.install: fix install paths
    for multiarch layout
  * d/p/0003-fix-ubuntu-paths-and-toolchain.patch: refresh patch

 -- Maintainer Name <maintainer@ubuntu.com>  Tue, 16 Jun 2026 08:00:00 -0300
